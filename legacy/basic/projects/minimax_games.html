Minimax Games

2018

See the source code on Github

This project was done as part of a school assignment aimed at teaching students the basics of recursion. The games are text-based two-player, sequential move, zero-sum, perfect-information games. The recursive/iterative strategies work by checking every possible move from the current state and determining whether each will result in a win or loss for the current player if both players play perfectly.

The assignment only required a recursive and iterative minimax function. However after seeing the inefficiency in regards to time, I created two more functions to optimize the strategies with some help from a Teaching Assistant.

Memoization

By storing states already seen, and their respective scores in a dictionary, I was able to skip states already "seen" and save the program from redundant repetitiveness.

Myopia

In this strategy I limited the recursion depth to 4, in essence, giving the strategy "nearsightedness" (a.k.a. Myopia). Once depth 4 was reached, I used the rough outcome strategy to approximate the state's score. This strategy significantly sped up the program however it did sacrifice accuracy in order to achieve this.
